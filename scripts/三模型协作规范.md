# 🎯 三模型协作规范与架构设计

## 📋 核心原则

1. **满血三模型**：必须同时使用3个大模型进行专业分工
2. **永不降级**：任何模型失败时必须有备用机制，不降级为简单逻辑
3. **专业分工**：每个模型有明确的专业领域和职责边界
4. **智能路由**：根据性能、成本、可用性动态选择最佳模型
5. **结果对比**：多模型并行分析，结果对比提升准确性

## 🏗️ 架构设计

### 三模型专业分工

| 模型 | 职责 | 专业优势 | 对应API | 模型类型 |
|------|------|----------|---------|----------|
| **庄模型** | 庄向证据分析 | 概率推理、历史模式识别 | OpenAI GPT-4o mini | 概率推理型 |
| **闲模型** | 闲向证据分析 | 逻辑严谨、规则分析 | Anthropic Claude Sonnet 4 | 逻辑严谨型 |
| **综合模型** | 综合决策 | 多源信息整合、风险评估 | Google Gemini 1.5 Flash | 整合决策型 |

### 4. 降级保障机制

```python
class FallbackManager:
    """降级保障管理器"""
    
    async def ensure_analysis(self, primary_task: Callable, fallback_tasks: List[Callable]):
        """
        确保分析任务完成的三级保障：
        1. 首选模型
        2. 备选模型
        3. 本地规则引擎
        """
        for task in [primary_task] + fallback_tasks:
            try:
                result = await task()
                if self._validate_result(result):
                    return result
            except Exception as e:
                log_error(f"模型失败: {e}")
                continue
        
        # 三级保障全部失败，返回保守决策
        return self._conservative_fallback()
```
