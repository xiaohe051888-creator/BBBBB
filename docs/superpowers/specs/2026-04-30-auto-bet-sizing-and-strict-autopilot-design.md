# 全自动托管闭环：指数下注金额 + 3AI/规则双模式 设计稿

日期：2026-04-30

## 目标

1. 运行闭环固定为：上传 → 自动分析预测下一局 → 自动下注（必须）→ 才允许开奖 → 开奖入路图 → 自动分析下一局循环。
2. 自动下注金额由“预测置信度”指数/加速映射到下注金额，并遵循 MIN_BET/MAX_BET/BET_STEP 与余额约束。
3. 3AI 模式下注后触发实时微学习（仅当前靴有效）；规则模式不触发微学习。

## 现状与差距

- 当前系统已在上传/开奖后异步触发分析，并在分析完成后自动调用下注服务；下注金额来源于分析结果的 `bet_amount`（规则/AI 内部给出），未统一按置信度指数映射。
- AI 调用异常时可能返回“观望”，目前用兜底方向“庄”保证必下，但下注金额策略仍需统一。

## 设计

### A) 指数下注金额（核心）

定义置信度 `conf ∈ [0,1]`，下注金额映射：

- 若 `conf < c0`：`amount = MIN_BET`
- 否则：
  - `x = (conf - c0) / (1 - c0)`
  - `amount = MIN_BET + (MAX_BET - MIN_BET) * x^γ`

参数（用户确认）：
- `c0 = 0.60`
- `γ = 2.0`

约束：
- 最终金额按 `BET_STEP` 向下取整
- 上限不超过 `MAX_BET`
- 若余额不足以覆盖最终金额，则下注金额取 `min(amount, balance)`；若 `balance < MIN_BET` 则进入“余额不足”（系统无法继续下注）。

落点：
- 下注金额由后端在 `run_ai_analysis` 内统一计算并落库/广播，上传与开奖触发下注直接使用该金额，避免多处分叉。

### B) 预测方向（必须下注）

- 若预测结果不是“庄/闲”，统一兜底为“庄”（保持现有行为，保证闭环不中断）。

### C) 微学习触发与“换靴遗忘”

- 仅当 `prediction_mode == "ai"` 且下注成功后触发微学习（沿用现有 `place_bet` 内触发点）。
- 微学习记忆 `AIMemory` 按 `boot_number` 隔离；新靴自然不会读取旧靴记忆；覆盖本靴会清空本靴记忆（保持现状）。

## 验收

1. 规则/AI 模式下，分析返回的 `bet_amount` 均来自指数映射策略（不是 AI 内部随意给值）。
2. 上传→分析→自动下注→开奖→下一局循环可回归（新增/更新测试）。
3. 规则模式不触发微学习；AI 模式下注后触发微学习（现有逻辑不回归）。
