# 前端全站移动端自适应设计

日期：2026-05-04

## 目标

- 对前端所有用户可见页面做移动端自适应适配。
- 仅调整页面排版、布局、间距、滚动、触控热区和信息呈现方式。
- 不改变任何业务功能、交互流程、接口协议和权限逻辑。

## 范围

- 页面：`DashboardPage`、`RoadMapPage`、`LogsPage`、`MistakeBookPage`、`BetRecordsPage`、`UploadDataPage`、`AdminPage`、`ModeSelectPage`
- 组件：表格、统计卡片、弹窗、头部操作栏、五路图容器、分析面板、日志详情
- 样式：`global.css`、`mobile.css` 与各页面现有内联布局

## 约束

- 手机竖屏优先，兼顾小平板。
- 只做布局适配，不改功能，不删功能，不增加新业务逻辑。
- 尽量复用现有类名和页面结构，避免大规模重构。

## 设计原则

### 一、统一移动端骨架

- 所有页面在手机宽度下统一使用：
  - 更紧凑但不拥挤的左右留白
  - 可换行的头部操作区
  - 更大的按钮点击区域
  - 统一底部安全区，避免被移动端底部导航遮挡

### 二、信息优先级重排

- 手机上优先显示结论、状态、关键按钮。
- 次级信息通过折行、纵向堆叠、卡片化方式展示。
- 尽量减少双列并排，必要时降为单列。

### 三、表格移动端卡片化

- 数据表在手机下优先转为纵向信息卡片，避免横向滚动。
- 若必须保留横向滚动，则限制在局部容器，不影响整页体验。

### 四、弹窗全宽与内容可滚动

- 弹窗在手机上接近全宽显示。
- 内容区域限制最大高度并支持内部滚动。
- 底部按钮在窄屏下可换行或竖排。

### 五、五路图与分析面板

- 五路图保持专业展示，但外层容器移动端允许顺滑横向浏览。
- 分析面板、状态栏、统计卡片在手机下改为单列堆叠，避免挤压。

## 页面级策略

### DashboardPage

- 左右双栏改为单列纵向流。
- 五路图、分析面板、记录表格依次纵向排列。
- 顶部状态栏、等待开奖栏和快捷按钮支持换行。

### RoadMapPage

- 顶部切换与刷新按钮允许换行。
- 图表页标题区和图例在手机下分行。
- 原始数据表与分析页卡片改为单列。

### LogsPage / MistakeBookPage / BetRecordsPage

- 筛选区改为多行流式布局。
- 表格维持移动端卡片化展示。
- 详情弹窗支持手机全宽和内部滚动。

### UploadDataPage / AdminPage

- 顶部工具栏、表单区、统计区和操作按钮改为单列优先。
- 多卡片区域在手机下强制单列。
- 文本域和输入框占满容器宽度。

## 验收标准

- 所有页面在手机宽度下无横向页面级溢出。
- 关键按钮不被遮挡，触控热区足够。
- 弹窗可完整浏览与操作。
- 表格、统计区、头部操作栏在手机下可读、可点、可滚动。
- 构建通过，主要页面无明显布局回归。
