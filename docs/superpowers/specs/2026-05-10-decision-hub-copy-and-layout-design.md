# 决策中枢文案与版式升级设计

## 背景

当前前端已经完成了中文化、深色未来感和基础分析卡/详情弹层改造，但在实际使用中仍存在两个明显问题：

1. 文案气质仍然偏“解释给新人听”，不够像成熟的智能决策系统
2. 详情页、分析卡、记录详情的排版层级还不够稳定，信息块之间的对齐、留白、节奏和按钮语义不够工整

用户已经明确提出新的目标：

- `智能分析` 和 `智能分析详情` 不要“大白话”
- 整体要更像“高级智能系统”
- `记录详情` 的标题体系也要更克制、更智能
- 不是只改几个词，而是连同 `UIUX`、`排版布局`、`对齐`、`按钮文案` 一起收口

本次设计的目标，就是把这三块界面统一升级为一套更完整的“决策中枢”视觉与语言系统。

## 目标

本次改造要同时完成四件事：

1. 把 `智能分析卡`、`智能分析详情`、`记录详情` 的文案从“解释型”升级为“决策型”
2. 把三类界面的排版、间距、层级、按钮动作统一到一套中密度中枢风格
3. 保留当前深色未来感基底，不走夸张花哨路线
4. 确保改动可被现有组件结构承接，不引入与业务逻辑无关的大改造

## 非目标

本次不做以下事项：

- 不改分析逻辑、预测逻辑、状态机逻辑
- 不改日志数据结构和后端接口字段
- 不重新设计整站导航或全局主题
- 不追求“量化交易终端”那种极端技术风格

## 设计方向对比

### 方案 A：局部替换标题词

做法：

- 只替换用户点名的几个标题和按钮
- 不调整版式层级

优点：

- 改动最小

缺点：

- 只能解决局部口语化
- 看起来还是“旧界面换了几个词”

结论：

不采用。

### 方案 B：决策中枢语系 + 中密度排版收口

做法：

- 统一文案气质
- 同时调整信息分区、留白、按钮语义、字段标题、标题层级
- 保持当前深色未来风格和页面主骨架

优点：

- 改动收益最大
- 风格统一
- 风险可控

结论：

采用方案 B。

### 方案 C：全面终端化重做

做法：

- 把分析卡和记录详情彻底做成“专业量化终端”

优点：

- 风格冲击力最强

缺点：

- 过度技术化
- 容易破坏现有已认可的设计基线

结论：

不采用。

## 术语体系

本次统一采用“决策中枢”语系，原则是：

- 少解释，多结论
- 少口语，多系统词
- 少冗余提示，多明确动作
- 少散乱标签，多分层结构

### 智能分析区域术语

- `本局建议` -> `本局决断`
- `判断方式` -> `决策机制`
- `把握程度` -> `决断强度`
- `查看这次怎么判断的` -> `展开决断详情`
- `最终为什么押这个方向` -> `最终决断依据`
- `已完成判断` -> `已完成研判`

### 详情动作术语

- `看完了，回到主面板` -> `返回`
- `收起详情` -> `收起`

不再保留类似：

- “如果这一页已经看完了，可以直接从这里……”
- “回到主面板继续看本局状态”

这种带明显解释腔的辅助句。

### 记录详情术语

- `这次发生了什么` -> `变动`
- `对当前使用有什么影响` -> `影响`
- `建议你接下来怎么做` -> `状态`
- `你可能会关心的信息` -> `关键信息`
- `复制通俗说明` -> `复制摘要`
- `一键复制下面这段通俗说明` -> `可复制本条摘要，用于转发或留存`
- `原始记录（高级信息）` -> `系统记录`

## 版式设计

### 1. 智能分析卡

当前 [AnalysisPanel.tsx](file:///workspace/frontend/src/components/dashboard/AnalysisPanel.tsx) 中的结果态信息太容易堆在同一层，导致用户看到：

- 顶栏、结论、标签、进度圈、说明文案同时争抢注意力
- 主结论强，但辅助信息没有形成稳定结构

本次改造成三段式结构：

1. 顶栏
   - 左侧：`智能分析`
   - 右侧：`已完成研判`
2. 决断区
   - 左侧：`本局决断` 与主结果 `庄 / 闲`
   - 右侧：两枚统一宽度的信息胶囊
     - `决策机制`
     - `决断强度`
3. 依据区
   - 单独说明面板承接简要依据

布局要求：

- 结果区和标签区必须左右分栏对齐
- 标签区宽度统一，不能出现长短不一导致的视觉歪斜
- 结果字符与标签信息之间保持稳定呼吸感，不拥挤也不漂

按钮要求：

- `展开决断详情` 独立成一行
- 不再使用过宽、过口语的长按钮文案
- 按钮下方仅保留一句极短状态描述，长度明显短于当前版本

### 2. 智能分析详情

当前智能分析详情实际内嵌在 [AnalysisPanel.tsx](file:///workspace/frontend/src/components/dashboard/AnalysisPanel.tsx) 的 `Drawer` 中，并非单独文件。

这次详情区做四段式重排：

1. 标题栏
   - 标题保留 `智能分析详情`
   - 右侧动作简化为 `收起`
2. 结论总览卡
   - 展示 `本局决断`
   - 展示 `决策机制`
   - 展示 `决断强度`
   - 展示一段简短结论摘要
3. 依据区
   - 标题改为 `最终决断依据`
   - 详细说明使用更窄的文字行宽和更稳定的段落间距
4. 底部动作区
   - 只保留主按钮 `返回`

布局要求：

- 结论总览卡内部全部左对齐
- 信息标签采用统一的胶囊尺寸和对齐方式
- 依据区标题与正文之间的垂直间距要明确，不再像长文案直接铺满
- 底部动作区不再出现整段指导句

### 3. 记录详情

当前 [LogsPage.tsx](file:///workspace/frontend/src/pages/LogsPage.tsx) 中的详情弹窗已经具备分区结构，但语气和视觉仍偏“解释弹窗”。

本次保留当前三块核心信息卡，但统一成如下语义：

- `变动`
- `影响`
- `状态`

结构分为：

1. 顶部元信息区
   - 严重程度
   - 类别
   - 结果
   - 置顶状态
2. 主标题区
   - 人话标题保留，但排版更聚焦
3. 摘要区
   - `复制摘要`
   - 摘要文本框
4. 三块核心状态卡
   - `变动`
   - `影响`
   - `状态`
5. 关键信息区
   - 列表方式展示人话字段
6. 系统记录区
   - 折叠展示原始记录

布局要求：

- 三块核心状态卡要使用统一的标题样式、边框强度、圆角和留白
- 摘要区与三块状态卡之间留出更清晰的层级间隔
- 操作按钮区文案统一，避免“全选/复制通俗说明/关闭”显得像工具页

## UIUX 规则

### 1. 标题层级

统一使用三层：

- 主标题：卡片/弹层名称
- 区块标题：`本局决断`、`最终决断依据`、`变动`、`影响`、`状态`
- 字段标题：`决策机制`、`决断强度`

禁止再出现：

- 主标题和字段标题视觉强度接近
- 大段辅助文案和正文标题同级

### 2. 间距系统

统一采用中密度中枢布局：

- 卡片内边距统一
- 区块与区块之间的垂直间距统一
- 标签组内部行距统一
- 按钮与正文之间保留明确呼吸区

目标是：

- 既不拥挤
- 也不松散
- 单屏可读性优先

### 3. 对齐规则

- 标签项采用统一宽度或最小宽度
- 结论与标签分栏时要稳定对齐
- 列表型信息采用统一左对齐，不做参差摆放
- 折叠面板标题与正文风格保持一致

### 4. 按钮语义

统一只保留三类动作：

- 展开：`展开决断详情`
- 收起：`收起`
- 返回：`返回`

日志详情动作统一为：

- `全选`
- `复制摘要`
- `返回` 或 `关闭`

其中优先推荐把详情类弹窗底部主按钮统一成 `返回`，减少界面语言分裂。

### 5. 辅助文案减量

本次所有面向用户的解释型辅助文案都要缩短。

规则如下：

- 能由标题表达的，不再补一句解释
- 能由状态标签表达的，不再写一整段引导
- 长句优先压缩成结果句

这条规则尤其适用于：

- 智能分析详情底部说明
- 智能分析卡按钮下方说明
- 记录详情复制区提示文案

## 风险与控制

### 风险 1：文案变高级后不够直观

控制方式：

- 采用“决策中枢”而不是“极端量化终端”语系
- 使用短词替代长句，不用冷门术语堆砌

### 风险 2：只改字，不改层级，效果不明显

控制方式：

- 把排版层级、分区、留白、按钮语义一并纳入改造范围

### 风险 3：详情抽屉逻辑与主卡耦合，修改容易混乱

控制方式：

- 实现时可以在 `AnalysisPanel.tsx` 内做小范围抽离
- 但不做超出本次目标的大重构

## 预期结果

改造完成后，用户看到的界面应具备以下特征：

- 文案不再像“解释给小白听”，而更像系统在输出结论
- 排版更工整，分区更清晰，移动端更稳定
- 按钮更克制，动作更明确
- `智能分析`、`智能分析详情`、`记录详情` 三者形成同一套语言和视觉体系

## 结论

本次采用“决策中枢语系 + 中密度排版收口”的设计方案。

它不是简单换词，而是把：

- 文案气质
- 信息层级
- 排版节奏
- 留白规则
- 按钮语义

统一提升到更成熟的智能系统界面表达上，同时保持现有深色未来感基础和业务稳定性不变。
