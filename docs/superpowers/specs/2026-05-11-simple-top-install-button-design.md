# 顶部极简安装按钮设计

## 1. 背景

上一版安装入口已经支持固定显示、安卓帮助说明和 iPhone 引导，但页面表现仍然偏重。

用户最新要求很明确：

- 只要一个简单的 `安装 App` 按钮
- 显示在顶部
- 不需要整块板块
- 不需要说明
- 不需要引导内容
- 已经使用 App 版本打开时不显示

因此本次设计是在现有安装入口基础上进一步做减法，把交互收敛到“顶部单按钮”。

## 2. 目标

本次设计只做以下事情：

- 在顶部显示一个简单的 `安装 App` 按钮
- 未安装时显示
- 已经以 App 版本打开时隐藏
- 点击按钮时直接尝试调用安装流程

## 3. 非目标

以下内容全部不再保留：

- 安装入口整块容器
- 副文案，例如“像 App 一样打开”
- 安卓帮助说明层
- iPhone 添加到主屏幕引导层
- “稍后”按钮
- 冷却隐藏逻辑

## 4. 最终设计

### 位置

按钮继续放在顶部区域：

- `WorkflowStatusBar` 下方
- `AdminAlertsBar` 上方

这样位置稳定，但视觉最轻。

### 形态

只渲染一个按钮：

- 文案固定为 `安装 App`
- 不再包裹为单独卡片或信息条
- 不展示额外说明文字

### 显示规则

显示条件：

- 当前不是 `standalone` 模式
- 当前不是已安装 App 的打开方式

隐藏条件：

- `window.matchMedia('(display-mode: standalone)')` 为真
- `navigator.standalone` 为真

### 点击行为

点击按钮时：

- 若存在 `beforeinstallprompt`，直接调用安装流程
- 若不存在 `beforeinstallprompt`，不再弹说明层

这意味着部分浏览器场景下按钮点击后可能没有额外反馈，这是本次“极简化”方案的接受范围。

## 5. 技术调整

### `InstallAppEntry`

收敛为只负责：

- 判断 `visible`
- 渲染一个按钮
- 点击后调用 `onInstall`

不再负责：

- 平台文案切换
- 安卓帮助层
- iPhone 引导层

### `useInstallPromptState`

收敛为只保留：

- `visible`
- `triggerInstall()`

不再需要：

- `platform`
- `guideVisible`
- `helpVisible`
- `openGuide` / `closeGuide`
- `openHelp` / `closeHelp`
- `dismiss`

### 删除组件

以下组件应从接线中移除，且若无其他用途可以删除：

- `InstallAppHelp`
- `InstallAppGuide`

## 6. 测试要求

需要更新的测试重点：

- 未安装时显示一个 `安装 App` 按钮
- 已安装时隐藏按钮
- 点击按钮时调用安装函数
- 首页顶部仍然正确挂载按钮
- 不再断言帮助说明层或引导层

## 7. 结论

本次采用最小化方案：

- 顶部只保留一个 `安装 App` 按钮
- 已安装时隐藏
- 去掉所有说明、引导和复杂视觉结构

这是当前最符合用户要求的实现方向。
