# All Visible Content Chinese Cleanup Design

**Date:** 2026-05-10

**Goal:** 把当前剩余的用户可见英文、字母编码和技术化文案继续清理到只剩自然中文，重点覆盖日志 JSON 导出内容和管理页中仍然可见的技术术语。

**Architecture:** 保持后端数据结构、接口字段和值、核心分析与下注流程不变，只改前端展示与导出层。以 `LogsPage.tsx` 的 JSON 导出为一条链，以 `AdminPage.tsx` 和 `beginnerCopy.ts` 的管理页可见文案为另一条链，统一收敛到面向普通用户和管理员都能直接理解的中文产品语言。

**Tech Stack:** React, TypeScript, Ant Design, Vitest, Vite

---

## 1. Problem Statement

虽然前面的分析卡、详情抽屉和运行记录详情已经完成了大部分中文化，但仍有两类用户可见内容没有彻底收口：

- `LogsPage.tsx` 的 `JSON` 导出仍然直接带出后端原始日志内容，用户下载后依旧会看到英文、事件码和技术串
- `AdminPage.tsx` 与相关文案映射中，仍然保留了 `AI`、`单 AI`、`single_ai`、`rule`、`三模型协作模式` 这类技术化或半技术化说法

用户的明确要求是：**所有用户能看到的地方都只需要中文**。这意味着不仅页面主视图要中文，导出内容和管理页文案也要继续收口。

---

## 2. Design Goals

### 2.1 所有用户可见内容继续中文化

本轮目标覆盖：

- 日志页面 `JSON` 导出
- 管理页所有用户可见模式名、说明文案、状态文案

### 2.2 不影响核心系统

- 不修改后端接口字段
- 不修改后端日志保存结构
- 不修改分析、下注、开奖、回补等核心流程
- 只改前端展示文案和前端导出结果

### 2.3 中文要像产品语言，不像技术翻译

不是简单把英文替换成字面中文，而是要改成用户一看就明白的话。

例如：

- `AI分析` -> `智能分析`
- `单 AI 模式` -> `智能判断模式`
- `三模型协作模式` -> `多路综合判断模式`
- `rule` / `rule_fallback` -> `规则辅助` / `备用判断`

---

## 3. Scope

### 3.1 本轮改动范围

- `frontend/src/pages/LogsPage.tsx`
- `frontend/src/pages/AdminPage.tsx`
- `frontend/src/utils/logHumanizer.ts`
- `frontend/src/utils/beginnerCopy.ts`
- 相关测试文件

### 3.2 本轮不改

- 后端原始日志 JSON 结构
- 数据库存储值
- 管理页底层配置字段名
- 管理接口真实请求参数

说明：前端仍然可以继续使用原始值与接口交互，但呈现给用户和导出的内容必须是中文化结果。

## 4. LogsPage JSON Export Design

### 4.1 当前问题

当前 `exportToJSON()` 仍直接导出后端原始日志数据，因此导出文件里仍可能包含：

- `LOG-MDL-003`
- `single_ai`
- `rule_fallback`
- `analysis timeout after 45.00s`
- 原始 `description`

这违背了“所有用户能看到的地方都只需要中文”。

### 4.2 目标设计

`JSON` 导出改成 **用户可读 JSON 导出**。

每条日志导出结构应以中文解释为主，例如：

```json
{
  "时间": "2026-05-10 12:38:55",
  "靴内局号": "23",
  "标题": "智能分析：系统已自动改用备用判断",
  "这次发生了什么": "智能判断这次没有及时给出稳定结果，系统已经自动改用备用判断继续完成下注。",
  "对当前使用有什么影响": "这次不会中断本局流程，系统已经继续给出最终下注决定。",
  "建议你接下来怎么做": "无需操作，等待本局开奖结果即可。",
  "更多信息": [
    { "名称": "事件", "内容": "备用判断接手" },
    { "名称": "类别", "内容": "系统运行事件" }
  ]
}
```

### 4.3 行为要求

- 导出的 `JSON` 不再直出原始 `event_code`
- 不再直出原始 `description`
- 不再直出英文异常文本
- 继续保持结构化导出，但字段名和字段值都使用中文

---

## 5. AdminPage Visible Copy Design

### 5.1 当前问题

管理页虽然主要面向管理员，但依然是用户可见页面，因此以下内容也需要继续中文化：

- `AI分析`
- `单 AI`
- `三模型协作模式`
- `single_ai`
- `rule`
- `AI学习`
- `AI大模型`

### 5.2 目标语言

统一改为更自然的中文产品话术：

- `AI分析` -> `智能分析`
- `单 AI` -> `智能判断`
- `三模型协作模式` -> `多路综合判断模式`
- `规则参考模式` -> `规则辅助模式`
- `AI学习` -> `系统学习优化`
- `AI大模型` -> `模型设置`

### 5.3 修改原则

- 只改可见文本
- 不改真正传给接口的 `mode`
- 不改后端数据库值
- 管理员仍能完成同样的配置与操作

---

## 6. Shared Copy Rules

### 6.1 统一术语

- `AI` -> `智能`
- `AI分析` -> `智能分析`
- `单 AI` -> `智能判断`
- `三模型协作` -> `多路综合判断`
- `rule` -> `规则辅助`
- `rule_fallback` -> `备用判断`

### 6.2 禁止继续面向用户出现的内容

- 英文模式名
- 英文错误提示
- 事件编码作为主阅读内容
- 原始技术串作为导出主字段

---

## 7. Testing

### 7.1 LogsPage

需要覆盖：

- `exportToJSON()` 导出的字段名为中文
- 导出内容不再包含 `LOG-MDL-003`
- 导出内容不再包含 `analysis timeout after 45.00s`
- 导出内容优先使用 `humanizeLog()` 的中文解释

### 7.2 AdminPage / beginnerCopy

需要覆盖：

- 管理页页面文案不再出现 `AI`、`single_ai`、`rule` 等用户可见技术词
- 模式名称映射已经切换为中文产品话术

### 7.3 回归

继续保证：

- 管理页各操作仍可点击
- 日志页导出仍可用
- 构建通过

---

## 8. Success Criteria

满足以下条件才算完成：

- 用户在页面和导出内容里都不再看到明显英文或技术串
- 管理页可见模式名与说明文案全部变成自然中文
- 日志 `JSON` 导出改成用户可读中文结构
- 不影响后端接口、分析链路和工作流核心逻辑
