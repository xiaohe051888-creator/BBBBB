# 深度学习与微学习完善设计（3AI/单AI 分模式隔离）

日期：2026-05-01  
范围：后端学习链路闭环 + 管理端体验完善 + 回归测试补齐  

## 背景与目标

系统目前支持三种预测模式：`ai`（3AI）、`single_ai`（单AI）、`rule`（规则引擎）。现状已具备：

- 等待开奖期间的“等待期微学习”（下注后触发）
- 深度学习（生成 `ModelVersion`）并按模式隔离版本

仍存在可完善点：

- 代码中存在“上一局微学习”的实现，但未被调用，属于悬空逻辑
- 深度学习内部引擎调用需与“必须配置 API 才能启动”的规则一致（3AI/单AI 各走可用引擎）
- 管理端需要更清晰的“版本按模式隔离”的可视化/筛选能力

目标：

1. 确保微学习仅在“下注后等待开奖期间”触发，且微学习策略只对当前靴有效（换靴清空）。
2. 深度学习在 3AI / 单AI 下都要求先配置对应 API Key 才能启动，且学习引擎调用策略与校验一致。
3. 管理端补齐“版本模式筛选”与模式提示，使可验收、可复盘。
4. 新增/补齐回归测试覆盖核心路径，确保改动可回归验证。

非目标：

- 改写核心下注/结算状态机
- 引入新的学习模型或第三方依赖

## 术语

- 等待期微学习：下注后、等待开奖期间触发，提炼当靴五路走势策略总结，写入 `AIMemory`。
- 深度学习：对一个靴或全库数据做学习分析，生成新的 `ModelVersion`（按模式隔离），产出对应的 `prompt_template`。

## 现状关键点（对齐）

### 数据隔离字段

- `ModelVersion.prediction_mode`：版本所属模式（`ai` / `single_ai`）。
- `AIMemory.prediction_mode`：记忆所属模式（`ai` / `single_ai`）。
- `GameRecord.prediction_mode`：该局预测时使用的模式（用于后续按模式筛数据）。
- `MistakeBook.prediction_mode`：该条错题所属模式（用于深度学习分析错误样本）。

### 深度学习配置校验

- `ai`：要求至少配置一个 3AI 的接口密钥（OpenAI/Anthropic/Gemini 任意一个）。
- `single_ai`：要求配置单AI模式（DeepSeek V4 Pro）的接口密钥。

## 方案概述（采用）

采用方案：**仅保留等待期微学习（下注后等待开奖期间）**，并确保其策略/记忆不跨靴复用（换靴清空）。

### 流程设计

1. 用户下注后（`ai`/`single_ai`）：触发等待期微学习（已有）。
2. 深度学习仅由管理员在管理员页面手动点击“启动深度学习”触发：当当前模式为 `ai` 或 `single_ai` 时可启动，生成对应模式的新版本（已有，补齐引擎调用策略一致性）。

约束：

- 不在靴结束时自动触发深度学习（不做“自动深度学习”）。

## 详细设计

### 1）等待期微学习（仅当前靴有效）

触发时机：每局下注成功后，进入等待开奖期间触发。

有效范围：

- 微学习产生的策略/记忆仅对当前靴有效；换靴后必须清空，避免跨靴污染与复用

换靴清理策略：

- 在进入新靴时清空上一靴的微学习记忆与错题：
  - 清空 `AIMemory`（微学习策略/记忆）
  - 清空 `MistakeBook`（错题本）
  - 删除范围需覆盖 `ai` 与 `single_ai` 两种模式

### 2）深度学习引擎调用策略一致性

目标：校验与实际调用一致，避免“校验通过但内部固定调用某一家导致失败”。

调用策略：

- `single_ai`：深度学习引擎调用走单AI接口（DeepSeek V4 Pro）。
- `ai`：按优先级选择可用引擎：
  1. Anthropic（若已配置）
  2. OpenAI（若已配置）
  3. Gemini（若已配置）
  4. 若都未配置则报错（与校验一致）

模式约束：

- 只有“配置了该模式的 API Key”时，该模式才允许启动深度学习，并会保存该模式的学习版本

触发来源：

- 仅来自管理员页面的“启动深度学习”按钮触发的管理员接口
- 其它链路（例如靴结束自动深度学习）不在本次完善范围内，且需要确保不会自动启动

### 3）管理端体验完善

- “模型版本管理”表格新增筛选：
  - 全部 / 3AI / 单AI
- “启动深度学习”弹窗展示当前模式：
  - `ai`：3AI模式
  - `single_ai`：单AI模式（DeepSeek V4 Pro）
  - `rule`：提示不支持并阻止打开弹窗

### 4）测试计划

新增/补齐自动化测试（后端）：

- 换靴（上传新靴/结束本靴）会清空 `AIMemory` 与 `MistakeBook`，确保微学习只对当前靴有效。
- 深度学习：
  - `ai` 模式未配置任何 3AI key -> 启动深度学习应返回明确错误
  - `single_ai` 未配置 single key -> 启动深度学习应返回明确错误
  - 引擎选择策略：在不同 key 配置组合下，能选到可用引擎并返回可解析 JSON（可用 mock/patch 验证）

前端：

- 构建通过（`npm run build`）
- 管理端筛选 UI 不影响既有功能

## 验收标准

- `ai` 与 `single_ai` 模式都能：
  - 等待期微学习写入记忆（已有）
  - 深度学习生成新版本并按模式隔离（已有，策略一致性完善）
- 管理端可清晰看到版本所属模式并可筛选
- 回归测试全部通过
