## 🔴 P0 - Critical 问题

### 1. AI分析功能完全失效

**现象**:
- 前端一直显示"AI三模型正在分析中..."
- API返回`has_data: false`
- `predict_direction`为`null`
- 系统状态卡在"分析中"或"等待下注"但无预测结果

**根本原因**:
- `.env`中配置的API密钥是假密钥（所有密钥相同：`sk-of-tbzTTqZAfVGqCphuORHuLwFHHTdJQmDCOaZkzYgxFmsmFSwxJdLgcMnkbopOlCvk`）
- AI服务调用失败但没有错误处理和反馈机制
- `asyncio.create_task`异步触发分析，异常被吞没

**影响**: 系统核心功能完全无法使用

**建议修复**:
1. 配置真实的AI API密钥
2. 添加AI服务错误处理和降级机制
3. 前端显示分析失败状态而非无限loading
4. 记录AI调用错误日志

---

### 2. 下注功能500错误

**现象**:
- 调用`/api/games/bet`返回`Internal Server Error`
- 后端抛出`sqlalchemy.exc.IllegalStateChangeError`

**错误日志**:
```
sqlalchemy.exc.IllegalStateChangeError: Method 'close()' can't be called here;
method '_connection_for_bind()' is already in progress and this would cause
an unexpected state change to <SessionTransactionState.CLOSED: 5>
```

**根本原因**:
- `game.py`第68-75行使用了`async with async_session()`上下文管理器
- 但`_place_bet`函数内部可能又创建了新的会话或事务冲突
- SQLAlchemy异步会话使用不当

**影响**: 无法完成下注，工作流中断

**建议修复**:
1. 检查`manual_game_service.py`中的`place_bet`函数
2. 确保不会嵌套使用会话上下文
3. 统一会话管理策略

## 🟡 P1 - High 问题

### 3. 下三路前端显示异常

**现象**:
- 后端返回的大眼仔路数据有红有蓝（第6局是蓝）
- 但前端截图显示大眼仔路只显示红色

**后端数据**:
```json
{
  "game_number": 6,
  "value": "蓝",  // 数据正确
  ...
}
```

**可能原因**:
- 前端`getPointColor`函数可能没有正确处理"蓝"值
- 或者canvas渲染时颜色映射有问题

**影响**: 用户无法正确看下三路走势

**建议修复**:
1. 检查`canvasRenderer.ts`中的颜色映射逻辑
2. 确保同时支持"红"/"蓝"和"延"/"转"格式

---

### 4. AI分析无错误反馈

**现象**:
- AI分析失败时前端永远显示"分析中..."
- 用户不知道发生了什么
- 没有重试或取消机制

**建议修复**:
1. 添加分析超时机制（如30秒）
2. 分析失败时显示错误信息和重试按钮
3. 提供手动跳过AI分析的选项
