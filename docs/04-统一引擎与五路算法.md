# 统一引擎与五路算法

## 文档目标
- 统一定义大路、珠盘路、大眼仔路、小路、螳螂路的计算规则与输出格式。

## 业务认知前提
- 单局开奖结果是随机事件，不存在固定长期必然规律。
- 五路在排列、布局、显示维度存在差异，会形成短暂可用的阶段性特征。
- 阶段性特征可能随时中断，模型必须动态判断而非固定套用单一路径。

## 输出要求
- 5路2D实时完整带血迹走势图
- 统一供前端、模型、数据库使用

## 走势图一致性要求
- 当前靴5条路的排版、布局、显示顺序、落点规则必须与目标开奖网站一致。
- 前端展示坐标体系与统一引擎输出坐标体系必须一一对应，禁止前端二次改写导致偏移。

## 错误标记映射规则
- 每次预测错误生成唯一错误编号 `error_id`。
- 同一个 `error_id` 在5条路上都必须有对应标记点，且局号与坐标位置可追溯。
- 错误标记仅在显示层叠加，不改变原始路单结构与颜色逻辑。

## 一、大路 (Big Road)

### 规则（100%匹配网站27）
1. 只记录**庄**和**闲**，和局跳过不记录（但占位计数）
2. 相同结果 → 纵向往下排列（同行）
3. 不同结果 → 换到新列第一行开始（换列）
4. **每组最多6个**：同一列最多6个相同结果
5. **超过6个时**：在第6个位置向右平行延伸（换列继续）
6. 和局处理：在上一局的格子上加绿点标记，不换列

### 显示
| 含义 | 颜色 | 形状 |
|------|------|------|
| 庄赢 | 🔴 红色 | 实心圆圈 |
| 闲赢 | 🔵 蓝色 | 实心圆圈 |
| 和局 | 🟢 绿色 | 小点标记(叠加) |

### 数据结构
```python
RoadPoint(game_number, column, row, value, is_new_column)
# value: "庄" / "闲"
```

## 二、珠盘路 (Bead Plate)

### 规则
1. **固定14列宽度** × **每列6行**
2. 从左上角开始，**从上到下**填满一列后换到下一列
3. 超过6行后自动换到右边下一列的顶部
4. 记录所有结果（包括和局）

### 显示内容
每个格子直接显示：庄(B)、闲(P)、和(T)

### 核心概念

三条派生路**不记录庄闲**，而是分析大路的**模式一致性**：

| 颜色 | 含义 | 英文术语 |
|------|------|---------|
| 🔴 红 (延) | 规律延续 / 模式一致 / 可预测 | Predictability / Consistent |
| 🔵 蓝 (转) | 规律转折 / 模式断裂 / 随机 | Chaos / Broken pattern |

### 参数定义

| 参数 | 定义 | 大眼仔 | 小路 | 螳螂路 |
|------|------|--------|------|--------|
| **k** (偏移量) | 向左回看的列数 | k=1 | k=2 | k=3 |
| **m** | 当前结果在大路的行号 (1-based, 从上到下) | - | - | - |
| **n** | 当前结果在大路的列号 (0-based, 从左到右) | - | - | - |
| **p** | 参考列(n-k)的图标总数 | - | - | - |

### 开始计算时机

| 派生路 | 偏移k | 从大路第几列开始画第一个点 | 需要的最小列数 |
|--------|-------|--------------------------|---------------|
| **大眼仔路** | 1 | 第2列 (col_idx=1) | ≥2列 |
| **小路** | 2 | 第3列 (col_idx=2) | ≥3列 |
| **螳螂路** | 3 | 第4列 (col_idx=3) | ≥4列 |

### 标准算法：两种情形判断法

这是全球百家乐电子路单和实体赌场通用的标准算法。

#### 情形A：新列出现（m=1，即大路开启了新的一列）

当最新一个结果导致大路开启了一个**新的列**时：

**判断方法**：比较新列左边紧邻的第1列 与 左边第(k+1)列 的**高度（长度）是否相等**

```
┌─────────────────────────────────────────────────────┐
│  大路网格示意（从右往左看）：                         │
│                                                     │
│  ... [左k+1列] [左k列] ... [左2列] [左1列] [新列★]  │
│         ↑              ↑                           │
│    比较这两个列的高度                              │
│                                                     │
│  若 高度(左1列) == 高度(左k+1列) → 🔴 红 (延)       │
│  若 高度(左1列) != 高度(左k+1列) → 🔵 蓝 (转)       │
└─────────────────────────────────────────────────────┘
```

**等价的"虚拟反转法"**（livedealer.org描述）：
- 当 m=1 时，先虚拟反转当前结果（庄↔闲），按情形B计算得到虚拟答案
- 再将虚拟答案的颜色反转 → 得到最终真实答案

**各路的具体比较对象**：

| 派生路 | k | 比较对象 | 判断 |
|--------|---|---------|------|
| 大眼仔路 | 1 | 左边第1列 vs 左边第2列高度 | 等→红 / 不等→蓝 |
| 小路 | 2 | 左边第1列 vs 左边第3列高度 | 等→红 / 不等→蓝 |
| 螳螂路 | 3 | 左边第1列 vs 左边第4列高度 | 等→红 / 不等→蓝 |

#### 情形B：延续当前列（m≥2，即大路在当前列向下添加）

当最新一个结果是在**当前列向下延伸**（不是开启新列）时：

**判断方法**：从当前位置向左移动k格，再向上移动1格，检查该位置**是否有值**

```
┌─────────────────────────────────────────────────────┐
│  大路网格示意：                                      │
│                                                     │
│      列(n-k)        当前列(n)                       │
│     ┌──────┐      ┌──────┐                         │
│  m-1│ ? ★  │ ←←←←←│ 新增★ │  向左移k格+向上移1格    │
│     ├──────┤      ├──────┤   检查?位置是否有值      │
│   m │  ●   │      │  ●   │                         │
│     └──────┘      └──────┘                         │
│                                                     │
│  若 ?位置有值 → 🔴 红 (延) — 同行有数据             │
│  若 ?位置无值 → 🔵 蓝 (转) — 同行无数据             │
└─────────────────────────────────────────────────────┘
```

**各路的具体检查位置**：

| 派生路 | k | 检查位置 | 判断 |
|--------|---|---------|------|
| 大眼仔路 | 1 | 当前位置→左1格→上1格 | 有值→红 / 无值→蓝 |
| 小路 | 2 | 当前位置→左2格→上1格 | 有值→红 / 无值→蓝 |
| 螳螂路 | 3 | 当前位置→左3格→上1格 | 有值→红 / 无值→蓝 |

### 派生路的自身排列规则

确定红/蓝之后，派生路自身的排列方式与大路**完全相同**：

1. **与前一个值相同**（红→红 或 蓝→蓝）→ 往下排列（同行）
2. **与前一个值不同**（红→蓝 或 蓝→红）→ 换到新列
3. 每组最多6个，超过6个时向右延伸

### 显示样式差异

| 派生路 | 红色标记 | 蓝色标记 | 特殊说明 |
|--------|---------|---------|---------|
| 大眼仔路 | 🔴 空心圆圈 | 🔵 空心圆圈 | ○ 形状 |
| 小路 | 🔴 实心圆圈 | 🔵 实心圆圈 | ● 形状 |
| 螳螂路 | 🔴 斜线 / | 🔵 斜线 \ | 同一格可画两条线 |

## 四、完整演算示例

### 示例：庄庄闲庄庄庄闲闲庄闲庄庄闲庄 (14局)

#### 步骤1: 构建大路

```
大路 (Big Road):
  列0: 庄 庄          (2个, 局1-2)
  列1: 闲            (1个, 局3)  ← 新列
  列2: 庄 庄 庄      (3个, 局4-6) 
  列3: 闲 闲        (2个, 局7-8)  ← 新列
  列4: 庄            (1个, 局9)   ← 新列
  列5: 闲            (1个, 局10)  ← 新列
  列6: 庄 庄          (2个, 局11-12)
  列7: 闲            (1个, 局13)  ← 新列
  列8: 庄            (1个, 局14)  ← 新列
```

#### 步骤2: 计算大眼仔路 (k=1)

从大路第2列(col_idx=1, 即局3)开始计算：

| 局号 | 大路位置(m,n) | 情形 | 判断过程 | 结果 |
|------|--------------|------|---------|------|
| 3 | (1,1) 新列 | A | col0高=2, 不存在col(-1)→fallback | **蓝(转)** |
| 4 | (1,2) 新列 | A | col1高=1, col0高=2 → 不等 | **蓝(转)** |
| 5 | (2,2) 延续 | B | (2-1=1, 2-1=1) → col1行1有值? col1只有行0→**无值** | **蓝(转)** |
| 6 | (3,2) 延续 | B | (2-1=1, 3-1=2) → col1行2有值? col1只有行0→**无值** | **蓝(转)** |
| 7 | (1,3) 新列 | A | col2高=3, col1高=1 → 不等 | **蓝(转)** |
| 8 | (2,3) 延续 | B | (3-1=2, 2-1=1) → col2行1有值? ✅有 | **红(延)** |
| 9 | (1,4) 新列 | A | col3高=2, col2高=3 → 不等 | **蓝(转)** |
| 10 | (1,5) 新列 | A | col4高=1, col3高=2 → 不等 | **蓝(转)** |
| 11 | (1,6) 新列 | A | col5高=1, col4高=1 → **相等!** | **红(延)** |
| 12 | (2,6) 延续 | B | (6-1=5, 2-1=1) → col5行1有值? col5只有行0→**无值** | **蓝(转)** |
| 13 | (1,7) 新列 | A | col6高=2, col5高=1 → 不等 | **蓝(转)** |
| 14 | (1,8) 新列 | A | col7高=1, col6高=2 → 不等 | **蓝(转)** |

**大眼仔路排列** (按大路规则: 相同往下, 不同换列):
```
转 | 转 转 蓝 | 延 | 转 | 转 转 蓝 | 转
```

### 常见误区

| 误区 | 正确理解 |
|------|---------|
| "奇偶列判断法"是标准算法 | ❌ 不是标准。标准是**两种情形法**（新列vs延续）|
| 只比较每列的第一个点 | ❌ 每个大路点都对应一个派生路点 |
| 红蓝代表庄闲 | ❌ 红蓝代表**模式一致性**，与庄闲无关 |
| 三条路的判断逻辑不同 | ❌ 逻辑相同，只是**参考列偏移k不同** |
