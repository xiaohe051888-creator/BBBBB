# 统一引擎与五路算法

## 文档目标
- 统一定义大路、珠盘路、大眼仔路、小路、螳螂路的计算规则与输出格式。

## 业务认知前提
- 单局开奖结果是随机事件，不存在固定长期必然规律。
- 五路在排列、布局、显示维度存在差异，会形成短暂可用的阶段性特征。
- 阶段性特征可能随时中断，模型必须动态判断而非固定套用单一路径。

## 输出要求
- 5路2D实时完整带血迹走势图
- 统一供前端、模型、数据库使用

## 走势图一致性要求
- 当前靴5条路的排版、布局、显示顺序、落点规则必须与目标开奖网站一致。
- 前端展示坐标体系与统一引擎输出坐标体系必须一一对应，禁止前端二次改写导致偏移。

## 错误标记映射规则
- 每次预测错误生成唯一错误编号 `error_id`。
- 同一个 `error_id` 在5条路上都必须有对应标记点，且局号与坐标位置可追溯。
- 错误标记仅在显示层叠加，不改变原始路单结构与颜色逻辑。

## 一、大路 (Big Road)

### 规则
1. 只记录**庄**和**闲**，和局跳过不记录（但占位计数）
2. 相同结果 → 纵向往下排列（同行）
3. 结果变化 → 换新列（从第0行开始）
4. 同一列最多6个点（第0-5行），超过后强制换列
5. **颜色含义**: 红=庄, 蓝=闲

### 数据结构
- 每个点: `(column, row, value)` 其中 value ∈ {庄, 闲}
- 列号递增，行号0-based

---

## 二、珠盘路 (Bead Road)

### 规则
1. 固定网格: **14列 × 6行**
2. 从左到右、从下到上依次填入所有结果（含和局）
3. 颜色不代表庄/闲，只表示原始结果
4. 填满14列后自动换到下一行

### 数据结构
- 每个点: `(column, row, value)` 
- column = (index % 14), row = (index // 14) % 6

---

## 三、下三路 (Derived Roads) — 大眼仔路 / 小路 / 螳螂路

### 概述
三条派生路均**从大路(Big Road)衍生而来**，排列规则与大路相同（同色往下，异色换列），但**颜色不表示庄/闲**！

### 颜色含义（统一）
| 颜色 | 符号 | 含义 |
|------|------|------|
| **红** | 延 / ● | 与前参照列**结构相同**（规律延续） |
| **蓝** | 转 / ○ | 与前参照列**结构不同**（规律转折） |

### 起始位置（何时画第一个点）

| 路名 | offset | 从大路第几列开始 | 原因 |
|------|--------|------------------|------|
| 大眼仔路 | **1** | 第2列 | 需要第1列作为参考 |
| 小路 | **2** | 第3列 | 需要第2列作为参考 |
| 螳螂路 | **3** | 第4列 | 需要第3列作为参考 |

> 如果大路总列数 ≤ offset，则该派生路无数据输出。

### ★ 核心判断规则：奇偶列法（澳门/拉斯维加斯标准）

派生路上的列位置使用 **1-based 编号**（第1列、第2列、第3列...），按奇偶分为两种判断逻辑：

#### 规则A — 奇数列（第1、3、5、7...列）

**比较内容**: 当前点的结果 vs 前一个点的结果（跳过和局，看原始庄/闲值）

| 条件 | 结果 | 颜色 |
|------|------|------|
| 当前结果 == 前一个结果 | 结构相同 → "延" | **红** |
| 当前结果 ≠ 前一个结果 | 结构不同 → "转" | **蓝** |

#### 规则B — 偶数列（第2、4、6、8...列）

**比较内容**: 当前列在大路中的**首行row** vs 参考列(当前列-offset)的**长度(行数)**

| 条件 | 结果 | 颜色 |
|------|------|------|
| current_row < ref_col_length | 参考列在同行有数据 → "延" | **红** |
| current_row >= ref_col_length | 参考列在同行无数据 → "转" | **蓝** |

### 判断流程图文字版

```
对于大路的每一列 i (从 offset 开始):
    1. 确定 reference_col = 大路[i - offset]
    2. 确定 current_point = 大路[i] 的第一个点
    3. 计算派生路列号 position = (i - offset + 1)  // 1-based
    
    IF position 是奇数:
        找到 current_point 之前的一个有效点 prev_point
        IF prev_point.value == current_point.value:
            → 红("延")   // 相同
        ELSE:
            → 蓝("转")   // 不同
    
    IF position 是偶数:
        ref_length = reference_col 的长度(点数)
        IF current_point.row < ref_length:
            → 红("延")   // 同行有值
        ELSE:
            → 蓝("转")   // 同行无值
    
    4. 按大路规则排列:
       IF 与上一个派生路点值相同:
         往下排 (row+1)
       ELSE:
         换新列 (col+1, row=0)
```

### 示例演算

假设大路如下（6列）:

```
列0: 庄(0) 庄(1) 庄(2)      ← 3个点, 长度=3
列1: 闲(0)                 ← 1个点, 长度=1
列2: 庄(0) 庄(1)            ← 2个点, 长度=2
列3: 闲(0) 闲(1) 闲(2)      ← 3个点, 长度=3
列4: 庄(0)                 ← 1个点, 长度=1
列5: 闲(0) 庄(0)            ← 2个点, 长度=2 (注意: 这是换列后的新列)
```

#### 大眼仔路计算 (offset=1, 从大路列1开始):

| 派生路列号 | 大路列 | 参考列 | 判断类型 | 比较 | 结果 |
|-----------|--------|--------|---------|------|------|
| **1**(奇) | 列1:闲(0) | 列0:庄×3 | 奇数列: 值对比 | 闲≠庄 | **蓝(转)** |
| **2**(偶) | 列2:庄(0) | 列1:闲×1 | 偶数列: row<len? | 0<1 ✓ | **红(延)** |
| **3**(奇) | 列3:闲(0) | 列2:庄×2 | 奇数列: 值对比 | 闲≠庄 | **蓝(转)** |
| **4**(偶) | 列4:庄(0) | 列3:闲×3 | 偶数列: row<len? | 0<3 ✓ | **红(延)** |
| **5**(奇) | 列5:闲(0) | 列4:庄×1 | 奇数列: 值对比 | 闲≠庄 | **蓝(转)** |

排列后: 转→延→转→延→转 （每列1行, 因为交替出现）
