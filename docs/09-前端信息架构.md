# 前端信息架构

## 页面分区
- 启动区
- 顶部关键信息卡区（当前局、预测局、工作流计时器）
- 智能分析区（庄模型、闲模型、综合模型）
- 实盘日志区
- 开奖记录区
- 下注记录区
- 5路走势图区
- 统计区
- 管理员入口区（启动页右上角登录按钮）
- 管理员页面（AI大模型板块、数据库存储板块）
- 系统健康分区
- 本靴进度条区
- 一键导出复盘包入口

## 5路走势图板块规范
- 必须展示当前靴5条路的完整走势图（大路、珠盘路、大眼仔路、小路、螳螂路）。
- 排版、布局、显示方式需与目标开奖网站保持一致，仅允许叠加错误标记层。
- 同一次预测错误在5条路上的标记必须一一对应，使用同一错误编号。
- 点击任一错误标记可联动高亮其余4条路对应位置与开奖记录中的对应局号。

## 交互原则
- 小白可见可懂
- 全中文

## 顶部关键信息卡规范
- 当前局：`当前局：XX局 庄/闲`，显示最新已开奖结果。
- 预测局：`预测局：XX局 庄/闲`，显示下一局预测结果。
- 局号关系：预测局号必须等于当前局号+1。
- 工作流计时器：每轮从0秒开始计时，到150秒封顶。
- 超时判定：计时达到150秒仍未完成“采集-清洗-分析预测-记录入库-决策-开奖结算”整轮流程，判定为流程异常。
- 超时动作：前端状态切为异常，实盘日志写入错误事件并记录异常环节与系统动作。

## 智能分析板块规范
- 板块名称：智能分析（放在首页首屏可见区域）。
- 展示对象：当前局分析过程与下一局预测结论。
- 详细实施：字段字典、文案模板、异常规则、验收样例统一以20号文档为准。
- 展示格式：
- 庄模型：`庄模型：因为大路{要点}、珠盘路{要点}、大眼仔路{要点}、小路{要点}、螳螂路{要点}，所以{庄向结论}`
- 闲模型：`闲模型：因为大路{要点}、珠盘路{要点}、大眼仔路{要点}、小路{要点}、螳螂路{要点}，所以{闲向结论}`
- 综合模型：`综合模型：因为{庄闲证据对比}+{五路冲突处理}，所以最终结论下局预测{庄/闲}`
- 一致性要求：综合模型“最终结论”必须与顶部预测局方向一致。
- 可读性要求：单条摘要建议30到80字；每个路由要点建议6到12字；优先短句，避免术语堆叠。
- 可解释性要求：五路要点至少3路必须有明确方向词（偏庄/偏闲/中性）。
- 细节交互：支持展开查看“关键信号、反向风险点、冲突处理说明”。
- 异常处理：任一模型输出缺失或冲突时，板块显示“分析异常”，并写入实盘日志P1告警。

## 实盘日志展示样式规范
- 颜色分级：信息使用蓝色、成功使用绿色、告警使用橙色、错误使用红色
- 日志排序：默认最新在上，保持自动滚动到最新
- 置顶规则：系统启动、停止、连续失准触发、异常退回、模型版本切换必须置顶显示
- 可读格式：主展示字段为时间、局号、事件、结果、简短说明
- 展开字段：桌号、靴号、来源模块、事件唯一键
- 自适应下注高亮：档位切换与金额变化日志使用强调样式，便于小白快速识别
- 关联跳转：点击日志可联动定位到对应下注记录或开奖记录
- 顶部快捷操作：支持一键复制日志、停止自动滚动、恢复自动滚动

## 日志优先级规则
- 本节仅做概览，优先级分级、前端动作、保留时长与升降级规则统一以20号文档“日志告警等级动作矩阵”为准。
- P1概览：资金与结算类高风险事件强提醒并置顶。
- P2概览：模型决策与策略切换类事件默认展示并短时置顶。
- P3概览：流程状态与心跳类事件可折叠展示。

## 日志分类规则
- 系统状态：启动、运行中、等待开奖、策略修正中、异常、停止。
- 操作记录：启动系统、停止系统、重新选桌、管理员登录、触发AI学习。
- 工作流事件：采集、清洗、分析预测、入库、决策、开奖结算、超时告警。
- 资金事件：下注扣款、开奖结算、异常退回。

## 日志区操作规范
- 支持按局号筛选
- 支持按事件类型筛选
- 支持仅看异常
- 支持一键清空当前视图（不删除数据库历史）

## 新增高价值可视化模块
- 系统健康分：由采集稳定性、模型稳定性、结算一致性组成，实时显示总分
- 健康分公式：`健康分 = 0.35×采集稳定性 + 0.35×模型稳定性 + 0.30×结算一致性`
- 采样窗口：默认最近500个有效结算局，窗口不足按当前全部有效结算局计算
- 告警阈值：`>=85`正常，`70-84`观察，`<70`告警并置顶显示
- 本靴进度条：显示当前局号进度与阶段状态，本靴预计50到70局仅作参考显示
- 一键导出复盘包：导出本靴日志、5路图、错题记录、版本号、下注结算流水
