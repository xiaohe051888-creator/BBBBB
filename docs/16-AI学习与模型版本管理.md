# AI学习与模型版本管理

## 功能目标
- 通过AI学习持续提升综合模型分析预测准确率。
- 对学习结果进行版本化管理，确保可追踪、可回滚。

## AI学习触发
- 触发入口：管理员页面 -> AI大模型板块 -> AI学习按钮
- 触发后综合模型使用数据库历史数据进行深度学习
- 学习完成后生成新的模型版本号并记录时间与摘要
- 当综合模型连续3局预测错误时，仅触发在线策略重评估，不触发离线AI学习任务
- 点击开始学习前，数据库历史有效局总量必须达到200局（上限遵循数据库1000局历史记录滚动口径）
- 学习启动后需展示预计完成时间、当前进度与阶段状态
- 学习完成后必须输出学习内容总结（样本范围、关键变化、指标变化）
- 学习任务执行中禁止重复触发学习（任务锁）

## 在线重评估与离线学习边界
- 在线重评估：自动触发，作用于当前运行策略参数（权重、阈值、档位），不中断每局下注。
- 离线AI学习：仅管理员手动触发，生成模型新版本并进入版本管理流程。
- 二者互斥：在线重评估执行期间不得自动拉起离线学习任务。
- 离线AI学习按靴执行：单次学习仅允许一个桌号+靴号的数据样本，不允许跨靴混合学习。

## 版本管理规则
- 最多保留5个模型版本
- 新版本生成后，如超过5个版本，按策略淘汰旧版本
- 每个版本记录：
  - 版本号
  - 创建时间
  - 训练数据范围
  - 学习后关键指标
- 版本切换冷却：新版本启用后，至少运行20个有效局才允许再次自动切换

## 与业务目标关系
- AI学习的唯一核心目标是提升下一局庄闲预测准确率。
- 学习策略应支持动态变化，避免固定僵化导致效果下降。
- AI学习不生成和局预测能力，学习目标仅限庄/闲方向预测优化。
