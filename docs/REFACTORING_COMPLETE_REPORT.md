# 全面单桌台（基于靴）重构总结报告
时间：2026-04-23

## 1. 核心变更背景
根据用户需求：“要彻底的重构，不是单纯的修补，旧的该删除就删除，该更新的就更新”，且“把系统驱动桌台选择和桌台逻辑，跟桌子有关的全部去掉，只有一个桌子，所以就不需要桌子方面的了，只有靴”。
我们对整个系统进行了大刀阔斧的重构，从底层数据库到上层UI彻底移除了多桌台（`table_id`）相关概念。

## 2. 数据库与后端改造
1. **模型重构**：删除了 `schemas.py` 中所有模型的 `table_id` 字段，移除了相关的复合唯一约束，将“桌台+靴”的概念扁平化为纯“全局+靴”。
2. **会话管理**：原 `table_id` 字典映射的 `ManualSession` 被改造为全局单例，由系统唯一持有。
3. **API与WebSocket**：移除了所有 API 接口中的 `table_id` 参数与路由中的 `/:table_id`。WebSocket 也不再按房间隔离，全系统广播统一状态。
4. **清理遗留数据**：直接移除了原本包含冗余桌台数据的旧 `baccarat.db`，实现了纯粹的新生。

## 3. 前端界面与路由重构
1. **移除启动页**：彻底删除了 `StartPage.tsx`，用户现在直接进入首页进行数据上传与流程驱动，无需再进行桌台选择。
2. **路由扁平化**：移除所有 `/:tableId` 路由设计，变为纯粹的 `/dashboard`。
3. **首页顶部新增联动按钮**：
   - **结束本靴按钮**：强制触发当前靴的深度学习流程（分析、总结），并导航至上传页面准备新靴数据。
   - **上传数据按钮**：可直接在当前靴进行数据覆盖上传，无论何种方式上传，都会清空 pending 下注、重置等待状态，重新开启工作流，绝不会产生“幽灵下注”。
4. **Hook重写**：系统移除了 `useGameState`、`useQueries`、`useWebSocket` 等 10 多个 Hook 中的 `tableId` 参数，重构了相关的 TanStack Query `queryKey`。

## 4. 三模型与业务规则加固
1. **永不降级**：彻底删除了三模型的 Fallback（降级）代码。任何模型（庄、闲、综合）出现 API 故障或超时，都会直接抛出异常并暂停系统（通过 WebSocket 推送前端红色告警），要求用户手动排查或等待恢复，**坚决不使用备用模型替代**，确保复盘数据和 AI 记忆的绝对准确性。
2. **环境彻底净化**：去除了所有遗留的爬虫环境变量（如 `LILE333_HEADLESS`）与相关代码。

## 5. 测试与验证
- `npm run lint` 验证通过，消除了所有因重构产生的 TypeScript 类型警告。
- `npm run build` 验证通过，打包无异常。
- 后端 `pytest` 测试修正了所有不再需要的 `table_id` 参数，运行正常。

## 结语
系统已完成从“多桌爬虫”向“单全局手动智能选模”的彻底进化，UI与逻辑完全贴合新一代工作流。
