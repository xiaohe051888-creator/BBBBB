“重置本靴”交互方案草图

目标：不换靴也能用一批新路单覆盖当前靴，并清空本靴的微学习/未完成数据，让系统从第 1 局重新开始验证。 这是“测试优先”的工作流，不改变默认的自动下注 + 手动开奖节奏。

方案 A（推荐）：重置本靴 → 弹窗粘贴路单 → 确认覆盖

1) 首页顶部按钮：重置本靴
低误触
放在“结束本靴”旁边，但视觉更弱（次要按钮）。仅管理员可见/可用。
2) 弹窗输入：粘贴新路单
支持 1~72 局
支持两种输入：一行一个（庄/闲/和），或用空格/逗号分隔。系统自动编号为第 1~N 局。
3) 二次确认：这会清空本靴数据
不可逆
清空范围（仅本靴）：GameRecord/BetRecord/MistakeBook/RoadMap/AIMemory 以及内存态 pending_bet、预测缓存、连续错误计数等。 余额是否保留由你决定（通常保留，便于对比策略）。
4) 覆盖完成 → 自动触发下一局分析
无缝回到自动模式
覆盖后，系统进入“分析中”，给出下一局预测并自动下注；页面出现“开奖按钮”，你录入下一局结果继续跑。
适合：频繁测试、数据纠错、回放某段路单
风险控制：二次确认 + 管理员权限
对外观影响小：不引入新页面
