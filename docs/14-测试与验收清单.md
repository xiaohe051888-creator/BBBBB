# 测试与验收清单

## 测试范围
- 功能正确性
- 数据一致性
- 准确率指标
- 前端可用性
- 自适应下注规则正确性
- 本文用于测试执行与量化判定，最终上线门禁以19号文档勾选为准

## 重点验收项
- 智能分析板块字段与文案模板必须符合20号文档定义。
- 每个有效下注局下注覆盖率为100%，无跳局下注记录
- 每局下注金额均满足最小10、最大10000、10的倍数
- 顶部必须展示“当前局：XX局 庄/闲（最新开奖）”与“预测局：XX局 庄/闲（下一局预测）”
- 预测局号必须始终等于当前局号+1
- 首页必须展示智能分析板块，且包含庄模型、闲模型、综合模型三部分
- 综合模型必须按“因为…所以…”格式输出最终结论并落到下局庄/闲
- 智能分析板块综合结论必须与顶部预测方向一致
- 庄模型与闲模型必须同时展示五路要点（大路、珠盘路、大眼仔路、小路、螳螂路）
- 庄/闲模型单条摘要需简洁易懂，长度建议30到80字
- 综合模型必须支持保守/标准/进取三档文案差异模板，并与当前策略档位一致
- 前端档位切换提示词必须匹配20号文档提示词库，不得出现口径漂移
- 场景化按钮文案必须匹配20号文档按钮文案包（启动成功、策略切换、异常恢复、重算成功）
- 确认弹窗、风险提示弹窗、恢复失败弹窗必须匹配20号文档弹窗文案包
- 空状态与引导文案必须匹配20号文档文案包（无数据、等待开奖、网络波动、AI学习锁定、洗牌等待）
- 系统日志关键词与固定短语必须匹配20号文档关键词规范包
- 系统日志事件编码必须匹配20号文档编码规范包并可按前缀筛选
- 日志告警等级动作必须匹配20号文档动作矩阵（P1/P2/P3）
- 系统日志保留与归档必须匹配20号文档策略包（7天/30天/永久）
- 日志导出复盘包必须匹配20号文档规范（目录结构、必备文件、摘要模板）
- 复盘结论必须匹配20号文档模板库（趋势误判、转折误判、置信过高、样本不足）
- 复盘评分必须匹配20号文档评分模板包（可解释性、稳定性、风险控制、修复效果）
- 复盘改进动作清单必须匹配20号文档模板包（问题、动作、负责人、观察窗口、通过标准）
- 复盘周报必须匹配20号文档周报模板包（本周问题、已做动作、达标情况、下周计划）
- 工作流计时器必须从0秒起计时并在150秒封顶
- 达到150秒未完成整轮流程时，必须生成超时错误日志并记录异常环节
- 连续3错后可自动切换保守档，恢复后可梯度回升
- 下注金额有可追溯依据摘要，实盘日志与下注记录一致
- 系统仅做仿真下注验证，不与目标网站发生真实下注交互
- 连续3错应对仅降档不可停注
- 新靴判定满足局号递减双确认，防止误判换靴
- 注单异常退回满足幂等约束，不存在重复退回
- 数据异常仅允许作为中间诊断态，终态必须回到已结算或异常退回
- AI学习仅在数据库历史有效局总量达到200局后可点击开始学习，学习中不可重复触发
- AI学习按靴执行，单次学习不得跨靴混样
- 自动选模遵守最少运行局数冷却，避免版本抖动
- 启动页选择桌子点击启动后，进入首页同时自动触发智能选模（若有可用模型版本）
- 管理员首次登录必须修改默认密码，未修改不得继续敏感操作
- 连续输错密码5次锁定10分钟，锁定与解锁事件必须写入实盘日志
- 系统健康分、本靴进度条与复盘包导出功能可用
- 实盘日志必须支持分类筛选（系统状态、操作记录、工作流事件、资金事件）
- 实盘日志必须支持复制、停止自动滚动、恢复自动滚动

## 量化阈值
- 统计窗口：最近500个有效结算局，窗口不足时按当前全部有效结算局统计。
- 日志字段完整率：100%（时间、局号、事件编码、事件类型、事件结果、说明、来源模块、事件唯一键必填）。
- 150秒超时漏报率：0%（超时即有错误日志）。
- 注单异常退回幂等违规数：0。
- 预测局号与当前局号关系错误数：0。
- 智能分析与顶部预测方向不一致错误数：0。
- 三档文案与策略档位不一致错误数：0。
- 档位提示词与当前策略档位不一致错误数：0。
- 场景化按钮文案与20号文档不一致错误数：0。
- 弹窗文案与20号文档不一致错误数：0。
- 空状态与引导文案与20号文档不一致错误数：0。
- 系统日志关键词与20号文档不一致错误数：0。
- 系统日志事件编码与20号文档不一致错误数：0。
- 日志告警等级动作与20号文档不一致错误数：0。
- 系统日志保留与归档策略与20号文档不一致错误数：0。
- 日志导出复盘包与20号文档不一致错误数：0。
- 复盘结论模板与20号文档不一致错误数：0。
- 复盘评分模板与20号文档不一致错误数：0。
- 复盘改进动作清单与20号文档不一致错误数：0。
- 复盘周报模板与20号文档不一致错误数：0。
- 系统健康分告警阈值口径错误数：0（>=85正常，70-84观察，<70告警）。
- 管理员安全规则与日志留痕不一致错误数：0。

## 边界用例
- 149秒：不得产生超时错误日志。
- 150秒：必须首次产生超时错误日志并置顶显示。
- 151秒：不得重复刷屏；同一事件唯一键在时间窗内仅保留一条超时错误日志。
- 5分钟退回：以服务器入库下注时间为基准，到点即触发，且不得与已结算并存。
- 智能分析边界：任一模型摘要缺失时必须显示“分析异常”并写入P1日志。
- 智能分析边界：任一模型五路要点缺少任意一路时，判定为摘要不完整并写入P1日志。
