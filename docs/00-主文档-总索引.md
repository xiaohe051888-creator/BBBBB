## 1. 文档说明

- 文档原则：
  - 一个主文档总控
  - 一个功能一个文档
  - 所有文档全中文
  - 变更必须同步更新主文档

## 2. 项目核心定义
- 核心能力：
  - 爬虫采集准确率
  - 统一引擎五路算法正确性
  - 三模型预测准确率
  - 前端用户体验
- 系统核心目的：
  - 分析预测下一局庄闲，持续提升预测准确率，准确率越高越好
- 智能能力：
  - AI学习：基于数据库历史数据深度学习并创建模型版本
  - 错题本：启动后运行中仅针对本靴历史错误复盘，不跨靴，持续修正策略
  - AI记忆模块：启动后自动智能选择最佳模型版本
  - 连续3局失准应对：触发策略重评估与风控动作
- 预测范围约束：
  - 三模型只分析庄/闲，不分析和局
- 跟注验证约束：
  - 每个有效下注局必须下注，下注仅用于系统可靠性验证
  - 每个有效结算局必须进入已结算或异常退回
  - 所有下注为仿真数字记录，不向目标网站执行真实下注
  - 每局下注金额智能自适应，且必须满足下注边界与步长约束
- 前端核心要求：
  - 仅前端可操作可观测
  - 实盘日志全中文、简短易懂、可定位问题
  - 顶部必须展示当前局结果与下一局预测结果
  - 首页必须有智能分析板块，庄/闲模型要按五路要点（大路、珠盘路、大眼仔路、小路、螳螂路）输出“因为…所以…”摘要，综合模型输出最终结论链路
  - 工作流计时器按轮从0到150秒，超时必须记录错误日志
  - 日志字段采用主展示+折叠展示双层口径，并使用事件唯一键防重复刷屏
  - 实盘日志支持分类筛选、复制、停止自动滚动与恢复自动滚动
  - 工作流角色化展示（卡通头像）
